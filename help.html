Why Avoid EXIF Data?

‚ö†Ô∏è EXIF Data Caveats

  • File Exchange Issues: EXIF data is often stripped when files are shared via email, cloud storage, or social media
  • Platform Inconsistencies: Different operating systems and applications handle EXIF data differently
  • Privacy Concerns: EXIF data can contain sensitive location information that persists across file transfers
  • Metadata Loss: File compression, conversion, or editing can corrupt or remove EXIF data
  • Cross-Platform Problems: EXIF data may not be preserved when moving between Windows, Mac, and Linux

Benefits of Hardcoding to Filename

‚úÖ Why Filename-Based Geotagging is Better

  • Permanent: Coordinates become part of the filename - they cannot be lost or stripped
  • Universal: Works across all platforms, file systems, and applications
  • Visible: Users can immediately see coordinates without special software
  • Transfer-Safe: Survives any file transfer, compression, or conversion
  • Searchable: Filenames can be searched and filtered by location

Required Files & Data

Input Files:

  • Media Files: Images (JPG, PNG, etc.) and videos with timestamp in filename
  • GPX Track: GPS track file with timestamps and elevation data
  • Filename Format: Must include ISO-8601 date/time (or a common camera timestamp) in the filename, e.g. 2025-08-04T13:27:44Z, 20250804_132744, 2025-08-04 13.27.44.

GPX Requirements:

  • Timestamps: Each track point must have accurate UTC time
  • Elevation: Optional but recommended for complete data
  • Density: 2-5 second intervals recommended for accurate matching

Output File Naming Scheme

üìÅ Output Filename Format:

noexif_media_ORIGINALNAME___LONGITUDE_LATITUDE___elev__ELEVATION___.jpg

Components Explained:

  • noexif_media_ - Prefix identifying the file as geotagged without EXIF
  • ORIGINALNAME - Original filename (without extension)
  • ___ - Triple underscore separator for easy parsing
  • LONGITUDE_LATITUDE - GPS coordinates with 6 decimal places
  • elev__ELEVATION__ - Elevation in meters (if available from GPX)
  • .jpg - All files converted to JPEG format for consistency

    • Example Output:

    noexif_media_202508041426___13.123456_47.654321___elev__923__.jpg

    Workflow

    1. Load Media: Select image/video files with timestamped filenames
    2. Load GPX: Upload GPS track file with timestamps
    3. Auto-Assign: System matches media timestamps with GPX track points
    4. Manual Adjust: Fine-tune coordinates by clicking on map if needed
    5. Download: Get ZIP file with all geotagged media using new naming scheme

    Benefits of This Approach

    • Future-Proof: Coordinates will never be lost, regardless of file handling
    • Platform-Independent: Works on any system without special software
    • Human-Readable: Coordinates visible in file manager, search results, etc.
    • Batch Processing: Easy to process multiple files with consistent naming
    • Archive-Safe: Perfect for long-term storage and archival purposes
